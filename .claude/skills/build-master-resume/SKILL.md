# build-master-resume — 마스터 이력서 누적 병합

`ingest-resume`가 파싱한 결과를 `data/master-resume.md`(모든 경력의 원본)에 누적한다. 마스터는 자르지 않고 최대한 풍부하게 유지한다.

## 절차

1. **기존 마스터 로드** — `data/master-resume.md`가 있으면 읽는다. 없으면 새로 만든다(아래 템플릿).
2. **병합** — 새 파싱 결과를 섹션별로 기존 마스터에 합친다.
   - **신규 항목**: 그대로 추가한다.
   - **중복 항목**(같은 회사·기간 또는 같은 프로젝트명): 두 서술을 비교해 더 풍부한 쪽을 기준으로 합치고, 한쪽에만 있는 디테일·수치는 빠짐없이 흡수한다. 정보를 잃지 않는 것이 원칙이다.
   - **상충하는 서술**(같은 항목인데 내용이 다름): 임의로 하나를 버리지 말고 둘 다 보존한 뒤 사용자에게 어느 쪽이 맞는지 확인한다.
3. **스킬 통합** — 스킬/기술스택은 중복 제거하되, 분류(Language/Backend/Frontend/DB/DevOps/기타)는 유지한다.
4. **저장** — `data/master-resume.md`에 덮어쓴다.
5. **요약 보고** — 추가된 항목, 병합된 항목, 확인 필요한 상충 사항을 사용자에게 보고한다.

## 마스터 템플릿

```markdown
# 마스터 이력서 — {이름}

> 모든 경력의 원본(master). JD별 맞춤 이력서는 여기서 선별·강조해 생성한다.

## 기본 정보
## 경력/직무
## 프로젝트
## 스킬/기술스택
## 학력/자격/기타
## 출처
```

## 원칙

- **무손실 병합**: 의심스러우면 버리지 말고 남긴다. 마스터는 재료 창고이므로 풍부할수록 좋다.
- 정량 성과·기술명은 절대 요약하며 누락하지 않는다.
- "출처" 섹션에 어떤 원본 파일에서 왔는지 기록해 추적성을 유지한다.
