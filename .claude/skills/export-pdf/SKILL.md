# export-pdf — 마크다운 → PDF 변환

마스터든 맞춤 이력서든, 마크다운 파일을 받아 PDF로 변환한다. 사용자가 PDF를 명시적으로 요청할 때만 동작한다.

## 입력

- 변환할 마크다운 파일 경로(예: `data/outputs/디스펙터_2026-05-21.md`).
- 경로가 없으면 가장 최근 생성된 `data/outputs/*.md` 또는 `data/master-resume.md` 중 어느 것인지 사용자에게 확인한다.

## 변환 방법 선택

환경에 설치된 도구를 확인한 뒤 가능한 방법을 사용한다. 우선순위:

1. **Pandoc** (있으면 우선) — `pandoc --version`으로 확인.
   - `pandoc input.md -o output.pdf` (PDF 엔진 필요: wkhtmltopdf 또는 LaTeX)
   - 한글 폰트 깨짐 방지를 위해 가능하면 `--pdf-engine=wkhtmltopdf` 또는 적절한 CJK 폰트 옵션을 지정한다.
2. **마크다운 → HTML → 브라우저 인쇄** — Pandoc이 없을 때.
   - 마크다운을 스타일링된 HTML로 변환(간단한 CSS: 한글 폰트, 여백, 불릿 스타일).
   - 브라우저 자동화(claude-in-chrome)로 HTML을 열고 PDF로 인쇄하거나, 사용자에게 브라우저에서 인쇄하도록 안내한다.
3. **Python 라이브러리** — `markdown` + `weasyprint` 등이 설치돼 있으면 사용.

설치된 도구가 하나도 없으면, 어떤 방법을 쓸지(예: Pandoc 설치, 브라우저 인쇄) 사용자에게 묻고 진행한다.

## 절차

1. 입력 마크다운 파일 확정.
2. 사용 가능한 변환 도구 탐지.
3. 변환 실행, 결과 PDF를 `data/outputs/`에 같은 이름(.pdf)으로 저장.
4. 한글이 깨지지 않았는지, 레이아웃이 적절한지 사용자에게 확인 요청.

## 원칙

- 변환은 형식만 바꾼다. 내용을 수정·요약하지 않는다.
- 한글 폰트 깨짐은 PDF 변환의 흔한 실패 지점이므로 반드시 확인한다.
