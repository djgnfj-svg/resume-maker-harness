# ingest-resume — 이력서 수집·파싱

이력서 파일을 읽어 마스터 스키마에 맞는 구조로 분해한다. 이 스킬은 파싱까지만 하고, 마스터 병합은 `build-master-resume`에 넘긴다.

## 입력

- `.md` 또는 텍스트: Read 도구로 직접 읽는다.
- PDF: Read 도구로 읽으면 페이지별 텍스트가 추출된다. 표·들여쓰기가 흐트러질 수 있으니 의미 단위로 재구성한다.
- 대화로 붙여넣은 내용: 그대로 파싱한다.

## 절차

1. **원본 보관** — 파일로 받았으면 `data/sources/`에 원본을 복사해 둔다(감사·추적용). 같은 이름이 있으면 덮어쓰지 말고 날짜 접미사를 붙인다.
2. **파싱** — 내용을 아래 4개 섹션으로 분류한다. 어디에도 안 맞는 정보(소개, 연락처 등)는 "기본 정보"로 모은다.
3. **정량 성과 보존** — "95% 개선", "9시간→30분", "주당 30시간 절감" 같은 수치는 반드시 그대로 살린다. 요약하며 버리지 않는다.
4. **출력** — 파싱 결과를 마스터 스키마 형식의 마크다운으로 정리해 `build-master-resume`에 전달한다.

## 마스터 스키마 (분류 기준)

- **기본 정보**: 이름, 연락처(GitHub/Email), 한 줄 소개
- **경력/직무**: 회사, 기간, 직책, 사용 기술, 핵심 업무·성과
- **프로젝트**: 프로젝트명, 역할(1인/팀), 사용 기술, 주요 개발·성과
- **스킬/기술스택**: 언어, 백엔드, 프론트엔드, DB, DevOps, 기타. 본문 곳곳에 흩어진 기술(예: 업무 설명에만 등장하는 Nginx, Cloudflare 등)도 빠짐없이 추출해 모은다.
- **학력/자격/기타**: 학력, 자격증, 수상, 발표, 교육 과정

## 경력 vs 프로젝트 구분

- 고용 관계(회사 소속, 급여)가 있으면 **경력/직무**
- 개인/사이드/프리랜서 결과물이면 **프로젝트**
- 애매하면 사용자에게 묻거나, 원본의 섹션 구분을 따른다.

## 주의

- 없는 정보를 채우지 않는다. 빈 섹션은 비워 둔다.
- 날짜는 원본 표기를 유지하되, "재직 중"·"진행 중" 같은 상태도 그대로 보존한다.
