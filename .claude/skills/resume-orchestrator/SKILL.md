# resume-orchestrator — 이력서 메이커 오케스트레이터

한 사람의 마스터 이력서를 누적 구축하고 JD에 맞춘 이력서를 뽑아주는 워크플로우를 조율한다.

**실행 모드:** 서브 에이전트 (단일 전문가 `resume-maker`가 스킬을 순차 호출). 작업 간 병렬·토론이 없어 팀 통신이 불필요하다.

## 디렉토리 규약

```
data/
├── sources/           # 입력 원본(PDF/MD) 보관
├── master-resume.md   # 누적 마스터
└── outputs/           # JD별 맞춤 이력서
```

## Phase 1: 컨텍스트 확인 (요청 유형 판별)

사용자 요청을 아래 4가지 흐름 중 하나로 분류한다:

| 요청 신호 | 흐름 |
|----------|------|
| 이력서 파일/내용 제공, "넣어줘/정리/분해" | **A. 마스터 구축** |
| JD 텍스트/URL, "이 공고로 뽑아줘" | **B. 맞춤 이력서** |
| "PDF로 변환/뽑아줘" | **C. PDF 변환** |
| "다시/수정/보완" + 기존 산출물 존재 | **D. 재실행** (해당 흐름만 부분 재수행) |

`data/master-resume.md` 존재 여부를 먼저 확인해 초기/누적을 구분한다.

## Phase 2: 흐름별 실행

`resume-maker` 에이전트(`model: "opus"`)를 호출하거나, 메인에서 직접 해당 스킬을 사용한다.

**A. 마스터 구축**
1. `ingest-resume` — 파일/내용 파싱, 원본을 `data/sources/`에 보관
2. `build-master-resume` — `data/master-resume.md`에 누적 병합
3. 추가/병합/상충 항목 요약 보고

**B. 맞춤 이력서**
1. 마스터 존재 확인 (없으면 A 먼저 안내)
2. `tailor-resume` — JD 분석 → 선별·강조 → `data/outputs/`에 마크다운 저장
3. 갭 리포트 + 선별 근거 보고

**C. PDF 변환**
1. `export-pdf` — 대상 마크다운을 PDF로 변환
2. 한글 깨짐·레이아웃 확인 요청

**D. 재실행**
- 해당 흐름만 부분 재수행. 마스터는 보존하고 사용자가 지정한 부분만 수정.

## 데이터 전달

- 파일 기반: `data/` 하위 약속된 경로로 단계 간 전달.
- 최종 산출물만 사용자에게 보고, 원본·중간물은 `data/sources/`에 보존(추적용).

## 에러 핸들링

- 파싱·변환 실패 시 1회 재시도, 재실패면 누락을 명시하고 사용자에게 보고.
- 상충 데이터는 삭제하지 않고 둘 다 보존 후 사용자 확인.
- 마스터에 없는 내용은 맞춤 이력서에 지어내지 않고 갭으로 보고.

## 테스트 시나리오

**정상 흐름**: 사용자가 PDF 이력서 경로 제공 → ingest → build-master → "마스터에 경력 2건, 프로젝트 2건 추가" 보고 → 이후 JD 제공 → tailor → `data/outputs/`에 맞춤 이력서 + 갭 리포트.

**에러 흐름**: JD만 주어졌는데 `data/master-resume.md` 없음 → tailor 진입 전 "먼저 이력서를 넣어 마스터를 만들어야 한다"고 안내하고 A 흐름으로 유도.
