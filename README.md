# 이력서 메이커 (Claude Code 하네스)

마스터 이력서(모든 경력의 원본)를 누적 구축하고, 채용공고(JD)를 받으면 그에 맞춰 선별·강조한 **맞춤 이력서**를 뽑아주는 Claude Code 하네스. 회사 지원 여부 스크리닝도 지원한다.

> 채팅에 아래처럼 **자연어로 말하면** 알맞은 스킬이 자동으로 실행된다. 명령어를 외울 필요 없음.

## 사용법

| 하고 싶은 것 | 이렇게 말하면 됨 |
|---|---|
| 📥 이력서 넣기 | "이 이력서 넣어줘" + 파일 경로(.pdf/.md)나 내용 붙여넣기 |
| 🧱 마스터 이력서 만들기/갱신 | "마스터 이력서 만들어줘" / "이 이력서도 합쳐줘" |
| 🎯 JD 맞춤 이력서 뽑기 | "이 JD로 이력서 뽑아줘" + 채용공고 텍스트나 URL |
| ✏️ 맞춤 이력서 수정 | "방금 이력서 다시 뽑아줘 / 보완해줘" |
| 📄 PDF로 변환 | "PDF로 뽑아줘" |
| 🏢 회사 거를지 판단 (직접 입력) | 회사명/도메인 줄줄이 붙여넣고 "이 회사들 지원할지 걸러줘" |
| 🔖 원티드 북마크 스크리닝 | (크롬에 원티드 로그인 상태) "원티드 북마크 찾아서 스크리닝해줘" |

## 구성

- **에이전트** `resume-maker` — 이력서 수집·파싱·맞춤·PDF 변환 담당
- **스킬** (`.claude/skills/`)
  - `resume-orchestrator` — 전체 작업 조율
  - `ingest-resume` — 이력서 파일 파싱
  - `build-master-resume` — 마스터 이력서 누적 병합
  - `tailor-resume` — JD 기반 맞춤 이력서 생성
  - `export-pdf` — 마크다운 → PDF
  - `screen-companies` — 회사 재무·인원 기반 ✅지원/❌거름 판정
- **자세한 동작 규칙·변경 이력**은 [`CLAUDE.md`](./CLAUDE.md) 참고

## 폴더 구조

```
.claude/
  agents/    에이전트 정의
  skills/    스킬 정의
data/        개인 데이터 (이력서·JD·생성물) — .gitignore로 깃 제외
docs/        설계 문서
CLAUDE.md    하네스 포인터 + 변경 이력
```

> ⚠️ `data/`에는 실명·이력서 등 개인정보가 들어가므로 깃에 올리지 않는다. 백업이 필요하면 별도로 관리할 것.

## 시작하기

Claude Code에서 이 폴더를 열고, 위 "사용법"대로 채팅에 요청하면 된다.
