# 이력서 메이커 하네스 — 설계 문서

작성일: 2026-05-21

## 목적

여러 이력서를 받아 한 사람의 "마스터 이력서"(모든 경력의 원본)를 누적 구축하고,
나중에 채용공고(JD)를 받으면 그에 맞춰 선별·강조한 맞춤 이력서를 뽑아주는
Claude Code 하네스를 만든다.

## 형태

Claude Code 하네스 — 전문 에이전트 1명 + 스킬 4개.

```
.claude/
├── agents/
│   └── resume-maker.md          # 이력서 전문 에이전트
└── skills/
    ├── ingest-resume/           # ① 이력서 수집·파싱
    ├── build-master-resume/     # ② 마스터 이력서 누적 병합
    ├── tailor-resume/           # ③ JD 기반 맞춤 이력서 추출 → 마크다운
    └── export-pdf/              # ④ (선택) 마크다운 → PDF 변환

data/
├── sources/                     # 입력 원본(PDF/MD) 보관
├── master-resume.md             # 누적되는 마스터 이력서(섹션 구조화)
└── outputs/                     # JD별 생성된 맞춤 이력서
```

## 범위 / 전제

- **한 사람 전용** — 본인(사용자) 한 명의 마스터 이력서만 관리.
- **입력**: `.md` 파일과 PDF를 경로/링크로 전달. (폴더 드롭도 허용 가능하나 기본은 경로 전달)
- **출력**: 기본 마크다운. PDF는 원할 때만 별도 스킬로 변환.

## 스킬 상세

### ① ingest-resume — 이력서 수집·파싱
- 입력: `.md` 또는 PDF 파일 경로.
- PDF는 Read 도구로 파싱, 마크다운/텍스트는 직접 읽음.
- 원본을 `data/sources/`에 보관.
- 경력·프로젝트·스킬·학력 등 섹션으로 정규화한 중간 결과를 build-master-resume에 넘김.

### ② build-master-resume — 마스터 누적 병합
- ingest 결과를 기존 `data/master-resume.md`에 **누적 병합**.
- 중복 경력/프로젝트는 감지해 합치고, 새 내용은 추가.
- 마스터는 "모든 경력의 원본"이므로 자르지 않고 **최대한 풍부하게** 유지
  (자잘한 디테일·수치·성과 포함).
- 마스터 섹션 스키마(구조화 마크다운):
  - **경력/직무**: 회사, 기간, 직책, 주요 업무·성과
  - **프로젝트**: 프로젝트명, 역할, 사용 기술, 성과/수치
  - **스킬/기술스택**: 언어, 프레임워크, 도구, 숙련도
  - **학력/자격/기타**: 학력, 자격증, 수상, 발표, 자기소개 등

### ③ tailor-resume — JD 기반 맞춤 이력서
- 입력: JD(텍스트 또는 URL).
- 흐름:
  1. JD 핵심 요구사항·키워드 분석
  2. 마스터에서 관련 경력·프로젝트·스킬을 **선별·강조** (순서 재배치, 강조)
  3. 마크다운으로 `data/outputs/`에 저장 (파일명에 JD/회사 식별)
- **무결성 원칙**: 마스터에 **있는 내용만** 사용. 없는 경력/스킬은 지어내지 않음.
  JD가 요구하지만 마스터에 없는 항목은 "갭"으로 사용자에게 별도 보고.

### ④ export-pdf — 마크다운 → PDF (선택)
- 사용자가 "PDF로 뽑아줘"라고 할 때만 동작.
- 마스터든 맞춤 이력서든 마크다운 파일을 받아 PDF로 변환.
- 변환 방식(Pandoc / 브라우저 인쇄 등)은 환경에 맞춰 구현 단계에서 확정.

## 에이전트

`resume-maker` — 위 4개 스킬을 조율하는 전문 에이전트.
사용자 요청("이 이력서 넣어줘", "이 JD로 뽑아줘", "PDF로")을 받아 적절한 스킬로 라우팅.

## 데이터 흐름

```
이력서(.md/PDF 경로) → ingest → build-master → master-resume.md (누적)
                                                      ↓
                              JD 입력 → tailor → 맞춤 이력서(MD)
                                                      ↓
                                      (선택) export-pdf → PDF
```

## 미결 / 구현 단계에서 결정
- PDF 변환 도구 선택 (Pandoc vs 브라우저 인쇄 등) — 환경 확인 후.
- 중복 병합 시 충돌(같은 항목 다른 서술) 처리 규칙의 구체화.
