# 이력서 기반 회사 발견(job-discovery) 설계

> 작성일: 2026-06-23
> 대상 하네스: 이력서 메이커
> 산출물: 새 스킬 `discover-jobs`

## 1. 목적

잡사이트(원티드·점핏·사람인·잡코리아)를 **능동적으로 탐색**해, 마스터 이력서
(`data/master-resume.md`)의 경력·기술과 **매칭되는 공고를 발견하고 그 사이트의
북마크/스크랩에 자동 추가**한다.

기존 `screen-companies`(경로 B)가 *이미 저장한* 공고를 재무 기준으로 거르는
수동적 필터링이라면, 이 기능은 한 단계 앞에서 *새 공고를 찾아 북마크에 모아두는*
능동적 발견이다. 재무 스크리닝은 이 기능이 직접 하지 않고, 북마크가 쌓이면
사용자가 기존 `screen-companies` 경로 B를 별도로 실행해 처리한다.

**책임 경계 (한 줄):** 발견 = 새 공고를 찾아 북마크 / 스크리닝 = 저장된 공고를
재무 판정. 두 기능은 명확히 분리한다.

## 2. 확정 요구사항

브레인스토밍에서 확정한 결정 사항:

| 항목 | 결정 |
|------|------|
| 검색 출발점(seed) | 마스터 이력서에서 **자동 추출** (사용자 키워드 입력 불필요) |
| 파이프라인 범위 | 검색 → 매칭 → **북마크까지**. 재무 스크리닝은 별도(경로 B) |
| 대상 사이트 | 원티드 · 점핏 · 사람인 · 잡코리아 (단계적 도입) |
| 매칭 방식 | 에이전트 정성 판단 — **높음/중간/낮음 등급 + 이유** |
| 하드 필터 | 경력 주니어~3년 이하 / 직무(백엔드·AI·Agent·풀스택) / 마감 안 지난 공고 |
| 북마크 기준 | **높음 + 중간** 등급을 북마크 (낮음은 제외) |
| 경로 B 연결 | **북마크까지만 하고 멈춤** (자동 연결 안 함) |
| 중복 방지 | 이미 북마크된 공고 + **이미 ❌거른 회사**(과거 스크리닝 기록)도 제외 |
| 결과 기록 | 채팅 요약 + **파일 저장** (`data/job-discovery/<날짜>-discovered.md`) |

## 3. 아키텍처

- **새 스킬** `.claude/skills/discover-jobs/SKILL.md` — `screen-companies`와 형제 구조.
- **사이트 어댑터(reference)**:
  - 1차: `references/wanted-discovery.md`, `references/jumpit-discovery.md`
  - 2차(1차 검증 후): `references/saramin-discovery.md`, `references/jobkorea-discovery.md`
  - 스킬 골격·매칭·필터·중복방지·리포트는 **처음부터 4사 공통**으로 설계하고,
    사이트별 어댑터만 점진적으로 추가한다.
- **단계적 도입 근거**: 가장 불확실한 부분은 "자동 북마크"(저장 버튼 클릭). 개발
  직군 친화적이고 마스터 이력서 매칭도가 높은 원티드·점핏에서 먼저 검증한 뒤
  사람인·잡코리아로 확장한다. (경로 B가 동일하게 단계적으로 확장된 전례 있음.)
- **의존성**:
  - 크롬 연결(해당 사이트 로그인 세션) — **필수 전제**.
  - `data/master-resume.md` 읽기 (검색 seed + 매칭 기준).
  - `data/company-screening/*.md` 읽기 (과거 ❌거른 회사 제외).
- **출력**: `data/job-discovery/<날짜>-discovered.md`.
- **등록**: `resume-orchestrator` 트리거 목록 + `CLAUDE.md` 변경 이력에 추가.

## 4. 데이터 흐름 (파이프라인)

```
1. 마스터 이력서 로드 → 검색 seed 자동 추출
   (직무: 백엔드 / AI·Agent / 풀스택  +  핵심기술: Python·FastAPI·Django·LangGraph …)
2. 크롬 연결·로그인 확인        (안 되면 중단·안내 — 발견은 크롬 필수)
3. 제외 목록 로드               (과거 ❌거른 회사 + 이미 북마크된 공고)
4. 사이트별 검색               (어댑터별: 키워드·직무·경력 필터 걸어 공고 후보 수집)
5. 하드 필터                  (경력 주니어~3년 / 직무 백엔드·AI·풀스택 / 마감 안 지남)
6. 중복 제거                  (이미 북마크 + 이미 거른 회사 빼기)
7. 매칭 평가                  (각 공고 ↔ 마스터 이력서 → 높음/중간/낮음 + 이유)
8. 북마크                    (높음+중간 → 사이트 저장 버튼 클릭)
9. 리포트                    (채팅 요약 + 파일 저장)
   → 멈춤. 재무 스크리닝(경로 B)은 사용자가 별도 실행.
```

## 5. 매칭 로직

- **입력**: 공고(직무·자격요건·우대·기술스택·회사) ↔ `data/master-resume.md`.
- **등급 기준**:
  - **높음** — 핵심 직무 일치 + 요구 기술 다수가 마스터에 있음 + 연차 적합.
  - **중간** — 직무 인접 또는 기술 일부 일치(일부 갭 존재).
  - **낮음** — 직무/기술 갭 큼 → **북마크 안 함**. 리포트엔 "낮음"으로 남겨 왜
    제외됐는지 보이게 한다.
- **매칭 이유**: 어떤 경력·프로젝트·기술이 매칭됐는지 1~2줄로 서술.
  **마스터에 없는 내용은 지어내지 않는다**(tailor-resume 원칙 그대로).

## 6. 사이트 어댑터 — reference에 정찰로 확립할 항목

경로 B reference를 만들 때처럼, 사이트별로 크롬 정찰해 아래 4가지를 확정한다:

1. **검색**: 키워드·직무 카테고리·경력 필터를 거는 검색 URL/API.
   (예상: 원티드=직무 카테고리+검색 API, 점핏=기술스택 태그 검색 API)
2. **결과 파싱**: 공고명·회사명·링크·**마감일**·경력요건·기술스택 추출 경로.
3. **자동 북마크**: 저장/스크랩을 거는 방식(버튼 클릭 or API). **가장 불확실** →
   1차(원티드·점핏)에서 먼저 검증.
4. **기존 북마크 조회**: 이미 저장된 공고 목록 — 경로 B reference
   (`wanted-bookmarks.md` 등)를 재사용한다.

## 7. 출력 형식

채팅 표와 `data/job-discovery/<날짜>-discovered.md`에 **동일 내용**을 저장한다:

| 사이트 | 회사 | 공고명 | 경력요건 | 마감일 | 매칭 | 매칭 이유 | 북마크 |
|--------|------|--------|----------|--------|------|-----------|--------|
| 원티드 | 예시㈜ | AI Agent 엔지니어 | 경력 1~3년 | 07-15 | 높음 | LangGraph 멀티에이전트(VoicePrep·MorningBrief) 직접 구축 경험 일치 | ✅ |
| 점핏 | 가나㈜ | 백엔드 개발자 | 주니어 | 상시 | 중간 | FastAPI·Django 일치, k8s 요구는 갭 | ✅ |
| 사람인 | 다라㈜ | 프론트 개발자 | 경력 2년 | 07-10 | 낮음 | 프론트 전용 — 직무 갭 | — |

- **요약 한 줄**: 발견 N건 / 북마크 M건(높음 X·중간 Y) / 제외 Z건(이미 거름·중복).
- **상한 고지**: 사이트·쿼리당 상한을 두되, 잘라낸 건수는 로그로 고지한다
  (조용한 누락 금지).

## 8. 에러 핸들링

- **크롬 미연결/로그인 아님** → 중단·안내. 발견은 크롬 필수이며 폴백 없음.
- **특정 사이트 검색 실패** → 그 사이트만 스킵, 나머지 사이트 계속.
- **자동 북마크 클릭 실패** → 해당 공고를 "북마크 실패"로 표기(리포트엔 남김),
  다음 공고로 진행.
- **alert/confirm 유발 클릭 금지**, 무한 재시도 금지(2~3회 실패 시 멈추고 보고).
- 마스터 이력서·검색 결과가 비면 그 사유를 안내한다.

## 9. 트리거 & 등록

- **트리거 예시**:
  - "이력서 기준으로 지원할 회사 찾아줘"
  - "잡사이트 훑어서 맞는 공고 북마크해줘"
  - "원티드·점핏에서 나한테 맞는 공고 찾아줘"
- `resume-orchestrator` 트리거 목록에 추가.
- `CLAUDE.md` 변경 이력에 등록.
- `screen-companies`와의 구분을 스킬 description에 명시(발견 ↔ 스크리닝 혼동 방지).

## 10. 단계적 구현 순서 (구현 계획 시 참고)

1. 스킬 골격(`SKILL.md`) — 4사 공통 파이프라인·매칭·필터·중복방지·리포트 정의.
2. 1차 어댑터: 원티드·점핏 검색→파싱→매칭→**자동 북마크** 정찰·확립.
3. 1차 검증(실제 발견·북마크 동작 확인).
4. 2차 어댑터: 사람인·잡코리아 추가.
5. `resume-orchestrator`·`CLAUDE.md` 등록.

## 11. 비목표 (YAGNI)

- 재무 스크리닝(적자·매출·인원 판정) — `screen-companies` 경로 B가 담당. 중복 구현 안 함.
- 맞춤 이력서 자동 생성 — `tailor-resume`가 담당. 발견 파이프라인에 포함 안 함.
- 정량 점수 루브릭(0~100점) — 에이전트 등급(높음/중간/낮음)으로 충분.
- 스케줄 자동 실행 — 우선 온디맨드. 필요 시 후속.
